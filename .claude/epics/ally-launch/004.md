# Task 004: Onboarding Experience

## Parallel Group: 2 (Core Features)
**Can run parallel with:** 005
**Depends on:** 002 (Auth)
**Blocks:** Nothing (but affects activation)

## Objective
Guide new users through setup to maximize activation and first-value-moment.

## Requirements

### Must Have
- [ ] Welcome screen with personalization
- [ ] Name/display name capture
- [ ] Timezone detection/selection
- [ ] First conversation prompt
- [ ] Skip option (don't force)
- [ ] Progress indicator

### Should Have
- [ ] Avatar selection/upload
- [ ] Interest/use-case selection
- [ ] Quick tour of features
- [ ] Sample prompts to try
- [ ] Mobile-optimized flow

### Nice to Have
- [ ] Personality quiz for AI tuning
- [ ] Import from other AI tools
- [ ] Invite friends flow
- [ ] Achievement for completing

## Technical Approach
- Store onboarding state in user profile
- Use stepper component for progress
- Persist partial completion (resume if interrupted)
- A/B test different flows

### Database Additions
```sql
profiles:
  + onboarding_completed: boolean
  + onboarding_step: integer
  + use_cases: text[]  -- selected interests
  + preferences: jsonb  -- extensible prefs
```

## Acceptance Criteria
1. New user sees onboarding after first login
2. Can complete full flow in <60 seconds
3. Skip button available on every step
4. Progress persists if user leaves
5. Completed users don't see it again
6. First message sends successfully

## Metrics to Track
- Onboarding start rate
- Step completion rates (funnel)
- Time to complete
- Skip rate per step
- 7-day retention by completion status

## Estimated Effort
- Design: 0.5 days
- Implementation: 1-2 days
- Testing/iteration: 0.5 days
- **Total: 2-3 days**

## Notes
- Keep it SHORT - every step loses users
- Defaults should be smart (detect timezone, suggest avatar)
- First message should feel magical (fast response, personalized)
- Consider gamification (progress bar, celebration on complete)
