# Task 007: Settings/Account Management

## Parallel Group: 3 (Extensions)
**Can run parallel with:** 006
**Depends on:** 002 (Auth), 003 (Stripe)
**Blocks:** Nothing

## Objective
Allow users to manage their account, preferences, and subscription.

## Requirements

### Must Have
- [ ] Profile editing (name, avatar)
- [ ] Email change (with verification)
- [ ] Password change
- [ ] Subscription management (via Stripe portal)
- [ ] Account deletion
- [ ] Logout / logout all devices

### Should Have
- [ ] Timezone settings
- [ ] Notification preferences
- [ ] Theme preference (dark/light/system)
- [ ] Data export
- [ ] Connected accounts (OAuth)
- [ ] API key management

### Nice to Have
- [ ] Two-factor authentication setup
- [ ] Session management (view active sessions)
- [ ] Usage statistics
- [ ] Referral program

## Settings Structure
```
Settings
├── Profile

├── Account

│   ├── Connected accounts

│   ├── Notifications

    ├── Export data

```

## Technical Approach

### API Routes
```
GET/PATCH /api/user/profile
GET/PATCH /api/user/preferences
POST /api/user/change-email
POST /api/user/change-password
DELETE /api/user (account deletion)
GET /api/user/export
POST /api/user/billing-portal (Stripe redirect)
```

### Account Deletion Flow
```
User requests deletion
        ↓
Confirmation email sent
        ↓
User confirms (link with token)
        ↓
Soft delete (30-day grace)
        ↓
Hard delete after 30 days
```

## UI Components
- Settings layout with sidebar navigation
- Form components for each section
- Confirmation modals for destructive actions
- Toast notifications for saves

## Acceptance Criteria
1. User can update profile info
2. Email change requires verification
3. Password change requires current password
4. Stripe portal opens for subscription management
5. Account deletion has confirmation + grace period
6. Data export generates downloadable file
7. Theme preference persists

## GDPR/Privacy Requirements
- [ ] Data export in standard format (JSON)
- [ ] Account deletion actually deletes data
- [ ] Clear privacy policy
- [ ] Cookie consent if applicable
