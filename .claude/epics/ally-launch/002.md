# Task 002: Signup/Auth Flow

## Parallel Group: 1 (Foundation)
**Can run parallel with:** 001, 003, 008, 009
**Depends on:** Nothing
**Blocks:** 004, 005, 007

## Objective
Implement secure, frictionless authentication that gets users to their first conversation as fast as possible.

## Requirements

### Must Have
- [ ] Email/password signup
- [ ] Email verification
- [ ] Social OAuth (Google, GitHub)
- [ ] Password reset flow
- [ ] Session management
- [ ] Protected routes
- [ ] Logout functionality

### Should Have
- [ ] Magic link login
- [ ] Remember me / persistent sessions
- [ ] Account lockout after failed attempts
- [ ] Password strength indicator
- [ ] Rate limiting on auth endpoints

### Nice to Have
- [ ] 2FA / TOTP
- [ ] Passkey support
- [ ] SSO for enterprise

## Technical Approach
- **Provider:** Clerk (recommended) or Auth0
- **Why Clerk:** 
  - Pre-built components
  - Webhook support for user events
  - Easy Stripe integration
  - Good DX

### Auth Flow
```
Landing Page → Sign Up → Email Verify → Onboarding → Chat
                 ↓
            Social OAuth → Onboarding → Chat
```

## Database Schema
```sql
-- If using custom auth (not recommended for MVP)
users:
  id: uuid PK
  email: string UNIQUE
  email_verified: boolean
  created_at: timestamp
  
-- Always needed (user profile beyond auth)
profiles:
  id: uuid PK
  user_id: uuid FK → auth.users
  display_name: string
  avatar_url: string
  timezone: string
  created_at: timestamp
  updated_at: timestamp
```

## Acceptance Criteria
1. User can sign up with email in <30s
2. Social OAuth works with 1 click
3. Email verification sends within 10s
4. Password reset works end-to-end
5. Sessions persist across browser close
6. Protected routes redirect to login
7. Auth state syncs across tabs

## Security Requirements
- [ ] HTTPS only
- [ ] Secure cookie settings (HttpOnly, Secure, SameSite)
- [ ] CSRF protection
- [ ] Input validation on all fields
- [ ] Rate limiting on auth endpoints
- [ ] Audit logging for auth events

## Estimated Effort
- Provider setup: 0.5 days
- UI implementation: 1-2 days
- Testing/edge cases: 1 day
- **Total: 3-4 days**

## Notes
- Clerk is worth the cost for speed to launch
- Can migrate to self-hosted later if needed
- Track signup source for attribution (UTM params)
