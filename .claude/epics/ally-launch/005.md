## Technical Architecture

### Message Flow
```
User Input → API Route → AI Provider → Stream → Client
                ↓
            Database (persist)
```

### AI Provider Integration
- Primary: Anthropic Claude
- Fallback: OpenAI GPT-4
- Consider: Local models for privacy tier

### Database Schema
```sql
conversations:
  id: uuid PK
  user_id: uuid FK → profiles
  title: string (auto-generated)
  created_at: timestamp
  updated_at: timestamp

messages:
  id: uuid PK
  conversation_id: uuid FK → conversations
  role: enum (user, assistant, system)
  content: text
  metadata: jsonb (tokens, model, etc)
  created_at: timestamp
```

### API Routes
```
POST /api/chat
  - Creates message, streams response
  - Body: { conversationId?, message }
  
GET /api/conversations
  - List user's conversations
  
GET /api/conversations/:id
  - Get conversation with messages
  
DELETE /api/conversations/:id
  - Delete conversation
```

## Acceptance Criteria
1. Message sends and response streams within 2s
2. Conversation persists across sessions
3. History loads in <1s
4. Works on mobile (responsive)
5. Markdown renders correctly
6. Code blocks have syntax highlighting
7. Can create new conversations
8. Can switch between conversations

## Performance Requirements
- First token: <500ms
- Full response: streaming (no wait)
- History load: <1s for 100 messages
- Optimistic UI updates
