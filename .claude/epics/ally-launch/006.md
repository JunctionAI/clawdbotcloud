# Task 006: Skill System

## Parallel Group: 3 (Extensions)
**Can run parallel with:** 007
**Depends on:** 005 (Core Chat)
**Blocks:** Nothing

## Objective
Enable extensible capabilities through a skill/plugin system that lets Ally perform actions beyond chat.

## Requirements

### Must Have
- [ ] Skill registry/manifest system
- [ ] Built-in skills (3 minimum for launch)
- [ ] Skill invocation from chat
- [ ] Skill output rendering
- [ ] Permission model for skills
- [ ] Error handling for skill failures

### Should Have
- [ ] Skill enable/disable per user
- [ ] Skill marketplace UI
- [ ] Skill usage analytics
- [ ] Skill rate limiting
- [ ] Skill versioning

### Nice to Have
- [ ] Custom user skills
- [ ] Skill API for developers
- [ ] Skill chaining
- [ ] Skill scheduling (cron)

## Launch Skills

### 1. Web Search
- Query the web for current information
- Return summarized results
- Cite sources

### 2. Calculator/Math
- Evaluate mathematical expressions
- Unit conversions
- Financial calculations

### 3. Weather
- Current conditions
- Forecast
- Location-based

### 4. Code Execution (Stretch)
- Run Python/JS in sandbox
- Return output
- Show errors clearly

## Technical Architecture

### Skill Manifest
```typescript
interface Skill {
  id: string;
  name: string;
  description: string;
  version: string;
  permissions: Permission[];
  parameters: ParameterSchema;
  execute: (params: any, context: SkillContext) => Promise<SkillResult>;
}
```

### Skill Invocation Flow
```
User message → AI decides skill needed → Extract parameters
                                              ↓
                                        Execute skill
                                              ↓
                                        Format result
                                              ↓
                                        AI incorporates in response
```

### Function Calling Integration
- Use Claude's tool use / OpenAI function calling
- Define skills as functions
- AI decides when to use

### Database Schema
```sql
skills:
  id: uuid PK
  slug: string UNIQUE
  name: string
  description: text
  manifest: jsonb
  enabled: boolean
  created_at: timestamp

user_skills:
  user_id: uuid FK → profiles
  skill_id: uuid FK → skills
  enabled: boolean
  settings: jsonb
  usage_count: integer
```

## Acceptance Criteria
1. AI can invoke skills naturally in conversation
2. Skill results render nicely in chat
3. Users can enable/disable skills
4. Failed skills don't crash conversation
5. At least 3 skills working at launch
6. Skill usage tracked for analytics
