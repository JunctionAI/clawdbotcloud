## Objective
Implement Stripe for subscription billing with support for free trial, upgrades, and self-service management.

## Requirements

### Must Have
- [ ] Product/price configuration in Stripe
- [ ] Checkout flow (redirect or embedded)
- [ ] Webhook handling for subscription events
- [ ] Customer portal for self-service
- [ ] Subscription status sync to database
- [ ] Usage limits based on tier

### Should Have
- [ ] Free trial period (7-14 days)
- [ ] Promo code support
- [ ] Invoice emails
- [ ] Upgrade/downgrade flows
- [ ] Payment method management

### Nice to Have
- [ ] Annual billing discount
- [ ] Team/org billing
- [ ] Usage-based pricing component
- [ ] Metered billing for API calls

## Pricing Structure (Proposed)
```
Free Tier:
- 50 messages/day
- Basic skills
- 7-day history

Pro Tier ($15/mo):
- Unlimited messages
- All skills
- Unlimited history
- Priority support

Team Tier ($25/user/mo):
- Everything in Pro
- Shared workspace
- Admin controls
- API access
```

## Technical Approach

### Stripe Products Setup
```
stripe products create --name="Ally Pro" 
stripe prices create --product=prod_xxx --unit-amount=1500 --currency=usd --recurring[interval]=month
```

### Webhook Events to Handle
- `checkout.session.completed` → Create subscription record
- `customer.subscription.updated` → Sync status changes
- `customer.subscription.deleted` → Handle cancellation
- `invoice.payment_failed` → Notify user, grace period
- `invoice.paid` → Confirm payment

### Database Schema
```sql
subscriptions:
  id: uuid PK
  user_id: uuid FK → profiles
  stripe_customer_id: string
  stripe_subscription_id: string
  status: enum (trialing, active, past_due, canceled)
  price_id: string
  current_period_start: timestamp
  current_period_end: timestamp
  cancel_at_period_end: boolean
  created_at: timestamp
  updated_at: timestamp
```

## Acceptance Criteria
1. User can subscribe via checkout
2. Subscription status reflects in app within 5s
3. Customer portal accessible from settings
4. Webhooks handle all lifecycle events
5. Canceled users retain access until period end
6. Failed payments trigger email + grace period

## Security Requirements

- [ ] PCI compliance (via Stripe Elements)
- [ ] Secure API key storage
