## Permission Justifications

When Chrome asks "Why do you need this permission?", use these justifications:

**storage**
```
We need storage permission to save your tracked leads locally on your device. This is core to the extension's functionality. All data is stored in Chrome's local storage and never leaves your device (free tier) or is encrypted when synced (Pro tier).
```

**notifications**
```
We need notifications permission to send you follow-up reminders. This is a key feature - the extension notifies you when it's time to follow up with a lead (e.g., 3 days after initial contact). Users can disable notifications in settings.
```

**alarms**
```
We need alarms permission to schedule periodic checks for follow-up reminders. This allows the extension to check once per hour if any leads need follow-ups, without keeping a background script running constantly (better for performance).
```

**host_permissions (https://*.linkedin.com/*)**
```
We need access to linkedin.com to detect when you send messages and extract the recipient's information (name, profile URL, company). The extension only observes your activity - it does NOT send automated messages or scrape data. It's read-only observation of your manual actions.
```

## Privacy Practices

**Data Usage Disclosure:**

```
This extension collects:
- Names of LinkedIn users you message
- Profile URLs of LinkedIn users you message  
- Message text you send
- Timestamps of when you send messages

This data is used to:
- Display your message history
- Send follow-up reminders
- Calculate analytics (e.g., conversion rate)

This data is:
- NOT sold to third parties
- NOT used for advertising
- Stored locally on your device (free tier) or encrypted in cloud (Pro tier)

We collect this data to provide the core functionality of the extension. Without collecting message data, the extension cannot track your outreach.
```
